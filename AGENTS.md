# fabrashion-mobile — agent guide

> **Expo has changed a lot. This is Expo SDK 57.** Read the exact versioned docs at
> https://docs.expo.dev/versions/v57.0.0/ before writing any Expo/native code, and prefer the
> project's Expo skills (`building-native-ui`, `native-data-fetching`) over memory.
>
> **Running on device:** the current **Expo Go** (installed from expo.dev) bundles SDK 57, so it
> opens this project for day-to-day JS development — just keep Metro running. A **dev build**
> (`expo-dev-client` + EAS Build) is still required for custom native modules Expo Go doesn't ship
> (e.g. the PhonePe SDK later). Standalone/preview APKs and Play Store builds are unaffected — EAS
> compiles a full native binary.

Customer mobile app (React Native + Expo + TypeScript). **Design docs are the source of truth in
`../plans/`** — read `plans/README.md`, `plans/04-frontend-structure.md`, and
`plans/11-phase-0c-mobile-scaffold.md`; backend contract in `plans/05-api-design.md`.

**UI is being rebuilt to the "ANDRÓ — Maison de Mode" design** — plan `plans/13-andro-redesign.md`;
the decoded, pixel-accurate reference is `design/andro-template.html` (match it exactly). The
Material Symbols glyph subset is generated by **`scripts/build-icons.py`** (needs `pip install
fonttools`) from `design/icon-names.txt` → the 3 subset TTFs in `assets/fonts/` + the glyph map
`src/theme/material-icons.ts`. To add an icon: add its name to `icon-names.txt`, run the script (it
downloads the Rounded variable font on first run), commit the regenerated fonts + map.

**Three role shells** (plan `plans/14-staff-shell.md`): the account's `role` (returned by the backend
after the same phone-OTP login everyone uses — never chosen at login) selects the shell in
`src/app/_layout.tsx`: CUSTOMER → `(tabs)`, STAFF → `(staff)` (Store Ops: Dashboard·Inventory·
Orders·Profile), ADMIN → `(admin)` (Overview·Catalog·Staff·Orders). Back-office logic lives in
`features/staff` + `features/admin`; shared bits in `components/backoffice/`.

## Status
**Phase 0c done:** production-shaped foundation + navigable app shell. **Auth is live (phone-OTP):**
login → `POST /auth/otp/request` → OTP → `/auth/otp/verify` → `setSession` (with role) → shell. In dev
the request returns `devCode` (prefilled on the OTP screen) so there's no SMS provider needed; on a
device set `EXPO_PUBLIC_API_URL` to your LAN IP. **Catalog, cart, checkout, addresses, and orders are
live** (server-backed; the shop auto-swapped off `catalog/preview.ts`). Checkout places a real order;
a `__DEV__` confirm stands in for the PhonePe capture (phase 4c). **The staff + admin shells are live

Confirm → Dispatch → Deliver. A `__DEV__` confirm stands in for the PhonePe trial charge (4c).
Staff Add Item is still presentational (product creation is the admin path).

## Architecture & conventions
- **Routes live under `src/app/`** (not a top-level `app/`), alias `@/* → ./src/*`. Only route and
  `_layout` files belong in `src/app/` — never co-locate components/utils there.
- **Feature-first** in `src/features/<name>/` (`schema.ts`, `api.ts`, `hooks.ts`, `store.ts`,
  `README.md`) mirroring backend modules. Keep each feature README current.
- **Server state → TanStack Query** (it *is* the cache; don't duplicate into Zustand). **Client/
  session state → Zustand** (`features/auth/store.ts`).
- **All HTTP goes through `src/api/client.ts`** (axios: injects the bearer token, does
  401→refresh→retry, normalizes errors to `ApiError` from `{ error: { code, message } }`). Paths in
  `api/endpoints.ts`.
- **Server is authoritative on money/stock/totals.** Money is integer **paise**; format only via
  `lib/money.ts` — never hand-format ₹ or compute totals on the client.
- **Styling is native — NO NativeWind / Tailwind / CSS.** Inline styles + semantic tokens from
  `src/theme`. Since the ANDRÓ redesign (`plans/13-andro-redesign.md`) the palette is a **FIXED,
  light-only** luxury theme, not the old adaptive `Color` API. Use `colors.*` (semantic), `spacing`,
  `radii`, `fontSize`; render text via the `Text` primitive (`serif`/`weight`/`size`/`track` props,
  Cormorant Garamond + Manrope) and icons via the `Icon` primitive (Material Symbols Rounded).
  **Never hardcode hex** — add a token to `theme/colors.ts`. No `useColorScheme()` dance (no dark mode).
- **Navigation:** a **custom `Tabs` bar** (`GlassTabBar`/`AndroTabBar`, not NativeTabs — the design's
  glass bar can't be done with native tabs). Customer: **Home · Categories · Wishlist · Cart ·
  Profile**; staff/admin shells pass their own tab map to `GlassTabBar`. Each tab is a route group
  with its own `Stack`; screens draw their **own headers** (`headerShown: false`). Auth + role gates
  = `Stack.Protected` in `src/app/_layout.tsx`. Cross-tab nav uses group-qualified hrefs
  (`/(staff)/(inventory)`); pushed sub-screens use their unique bare name (`/update-stock`). **Typed
  routes are on** — after adding routes, run the dev server (or it regenerates `.expo/types`) before
  `tsc`, else new hrefs error.
- **Storage:** tokens in `expo-secure-store` (`lib/secure.ts`); non-sensitive KV in
  `expo-sqlite/kv-store` (`lib/storage.ts`). No MMKV (needs a dev build), no AsyncStorage.
- **Config:** only `process.env.EXPO_PUBLIC_*` (via `src/env.ts`) — no secrets in the bundle.
- Filenames **kebab-case**; prefer `@/` alias imports; `expo-image` for images and SF Symbols.

## Commands
- `npx expo start` — dev server (Expo Go). `npx tsc --noEmit` — typecheck.
- `npx expo export --platform ios` — verify the whole graph bundles.
